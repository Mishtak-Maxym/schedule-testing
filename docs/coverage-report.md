# Coverage Report

## Модуль

Перегляд розкладу групи, трек JavaScript/Jest.

## Що було протестовано

- `frontend/src/helper/disableComponent.js`
- `frontend/src/helper/reduxFormer.js`

## Команди для запуску

```bash
cd frontend
npm install
npm test
npm test -- --coverage
npm run stryker
```

## Загальне покриття

Очікуваний результат після запуску тестів:

- Statements: 90%+
- Branches: 90%+
- Functions: 90%+
- Lines: 90%+

## Аналіз покриття

Найкраще покриті helper-функції, які відповідають за:

- перевірку порожніх значень;
- блокування або розблокування кнопки збереження семестру;
- пошук викладача або предмета за id;
- збереження вибраного викладача;
- фільтрацію розкладу для парного та непарного тижня;
- фільтрацію занять за викладачем.

Додані тести покривають позитивні сценарії, негативні сценарії та edge cases: `null`, порожній рядок, порожній масив, порожній об'єкт, невідомий id, неіснуючий тип тижня.

## Mutation testing

Для mutation testing додано файл:

- `frontend/stryker.config.mjs`

Мутації аналізують helper-файли:

- `src/helper/disableComponent.js`
- `src/helper/reduxFormer.js`

Очікувано тести мають вбити мутантів, які змінюють:

- `true` на `false`;
- `===` на `!==`;
- перевірки `null` / `undefined`;
- умови для парного та непарного тижня;
- фільтрацію за id.

## Скріншоти

Після локального запуску потрібно додати сюди скріншоти:

1. Результат `npm test -- --coverage`.
2. Результат `npm run stryker` або HTML-звіт `frontend/reports/mutation/mutation.html`.

## Висновок

Було створено повний набір unit-тестів для helper-функцій модуля перегляду розкладу групи. Тести згруповані через `describe`, використовують AAA-патерн і перевіряють як основні сценарії, так і граничні випадки. Також додана конфігурація Stryker для mutation testing.
